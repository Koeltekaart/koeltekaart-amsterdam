# Domain setup — koeltekaartamsterdam.nl

Two parts:
- **Part A — Registration** (for the partner): buy the domain. No technical knowledge needed, ~15 min.
- **Part B — Connection** (for Nojus / the technical owner): point the domain at the Azure-hosted site.

The two are deliberately separate: the partner only registers and pays; Nojus does the wiring.

---

## Part A — Register the domain (partner)

### Which registrar
Use **TransIP** (https://www.transip.nl) — a well-established Dutch provider, clear interface, ~€10/year for a `.nl` domain. A cheaper alternative is **Versio** (https://www.versio.nl). If the municipality has its own preferred registrar or IT procurement, that's fine too — **any `.nl` registrar works**; the only requirement is that Nojus can later manage the domain's DNS records.

### Steps
1. Go to **https://www.transip.nl**.
2. In the domain search box, type **`koeltekaartamsterdam.nl`** and search. It should show as **available**. Add it to your cart.
3. **Decline all extras.** The registrar will offer web hosting, email, "SSL certificates", website builders, etc. **We don't need any of these** — the website is already hosted elsewhere. Order **only the domain name**.
4. **Create an account** to complete the purchase. Use:
   - your **work email address**, and
   - the **organisation name** (GGD / Gemeente Amsterdam) as the registrant/owner.

   This matters: it makes the domain belong to the organisation, not to a personal account — important for the long term.
5. **Pay** (~€10 for one year; you can choose auto-renew so it doesn't expire).
6. **Done.** ✅

### One last thing: give Nojus access
So Nojus can connect the domain to the website, he needs to manage its DNS settings. Either:
- **Add him as a user/contact** in the TransIP account (Settings → Users), **or**
- **Share the login** with him securely (e.g. via a password manager, not plain email).

That's all that's needed from the partner. The rest is technical and handled by Nojus.

## Part B — Connect the domain to the site (Nojus)

> You can't do this until Part A is done and you have DNS access. The exact record
> values are generated by Azure when you add the domain — Claude will hand you the
> precise values to paste once you're at this step.

The site lives on Azure Static Web Apps. Plan:

1. **Azure portal → your Static Web App → Custom domains → Add.**
   - Add **`www.koeltekaartamsterdam.nl`** first (subdomains are the easy case).
   - Azure shows a **CNAME** target like `ambitious-bay-01e7c5c03.7.azurestaticapps.net`.
2. **In TransIP DNS** (Domains → koeltekaartamsterdam.nl → DNS):
   - Add a **CNAME** record: name `www`, value = the Azure hostname from step 1.
   - Azure may also ask for a **TXT** record to prove ownership — add it exactly as shown.
3. **The bare domain** (`koeltekaartamsterdam.nl`, without `www`):
   - Simplest, works on TransIP: set up a **URL redirect / "doorsturen"** from the bare domain to `https://www.koeltekaartamsterdam.nl`. So `www` is the canonical address and the bare domain bounces to it.
   - (Alternative if you want the bare domain to serve directly: move DNS to a provider that supports apex aliasing — e.g. Cloudflare DNS, free, with CNAME flattening — then add the apex per Azure's instructions.)
4. **HTTPS is automatic.** Azure issues and renews the TLS certificate for the custom domain — nothing to manage.
5. Wait for DNS to propagate (minutes to a couple of hours), then visit the domain to confirm.

### Notes
- Until the domain is live you can keep sharing the `*.azurestaticapps.net` URL.
- **Ownership stays clean for handoff:** because the domain is registered to the organisation and the site deploys from the `Koeltekaart` GitHub org, the municipality can take over hosting later by re-pointing DNS — no domain transfer needed.
